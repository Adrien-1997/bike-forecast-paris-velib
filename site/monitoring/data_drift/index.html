    Drift des données (Data Drift)¶

    Cette page documente la stabilité des distributions des entrées (features) et signale les changements de régime susceptibles de dégrader les performances du modèle.

    1) Définitions rapides¶

    • Data drift : les entrées (X) changent de distribution entre une période de référence et la période courante (ex. nouveaux usages, météo anormale, vacances scolaires).
    • Concept drift : la relation entre X et la cible (y) change (ex. nouveaux comportements clients qui cassent les patterns appris).
    • Référence vs. Courant : par défaut, on prend ~30 % début de l’historique en référence et ~30 % fin en courant (paramétrable).

    2) Indicateur principal : PSI¶

    Le Population Stability Index (PSI) mesure l’écart entre les histogrammes référence et courant :

    • Lecture (règle pratique) :
    • PSI < 0.1 : stable
    • 0.1 ≤ PSI < 0.2 : à surveiller
    • PSI ≥ 0.2 : dérive matérialisée (review requise)

    3) Santé des données (qualité)¶

    Avant de parler drift, on vérifie la qualité : - Manquants (montée soudaine = problème d’ingestion/sources)
    - Valeurs hors plage (capées, aberrantes)
    - Constantes (feature devenue fixe ⇒ non informative)

    4) Lecture analytique¶

    Pour chaque feature à PSI élevé : 1. Nature de la feature (météo, temporelle, station-level…)
    2. Direction du changement (moyenne, dispersion, asymétrie)
    3. Impact attendu sur la prédiction (via importance/élasticité)
    4. Segments touchés (clusters de stations, arrondissements, heures)

    Bon réflexe : croiser avec l’erreur par bucket (taux d’occupation) et les métriques par horizon.

    5) Actions et remédiations¶

    • Drift météo/contexte : enrichir features (pluie/vent réels, jours fériés locaux).
    • Drift horaire/hebdo : recalibrage de variables temporelles (lags/saisonnalités).
    • Drift station-level : réviser la normalisation par capacité, flagger les stations anormales (travaux/pannes).
    • Drift global persistant : réentraînement anticipé, mise à jour des fenêtres et seuils.

    6) Alternatives & compléments métriques¶

    • KS test (Kolmogorov–Smirnov) sur variables continues.
    • Wasserstein distance (earth mover’s distance) pour sensibilité aux déplacements de masse.
    • KL divergence (si distributions non nulles et lissées).
    • PSI par segment (cluster, arrondissement) pour isoler l’origine du drift.

    7) Politique d’alerte¶

    • Alerte jaune : ≥1 feature avec 0.1 ≤ PSI < 0.2 sur 3 jours glissants.
    • Alerte rouge : ≥1 feature avec PSI ≥ 0.2 ou >5 features en jaune.
    • Réponse : analyse ciblée + décision (feature engineering, réentraînement, exclusions temporaires).
